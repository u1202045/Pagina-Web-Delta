    Protocolos de Comunicacion:

    arduino maneja diferentes tipos de comunicación serial entre ellos se manejan los siguientes:

  • UART: (recepción-transmisión asíncrona universal) es uno de los protocolos serie más utilizados. La mayoría de los microcontroladores disponen de hardware UART. Usa una línea de datos simple para transmitir y otra para recibir datos. Comúnmente, 8 bits de datos son transmitidos de la siguiente forma: un bit de inicio, a nivel bajo, 8 bits de datos y un bit de parada a nivel alto.A diferencia de los otros tipos de comunicación la UART es asíncrona mientras que el resto es sincron y depende de un reloj.
  • SPI: Es otro protocolo serie muy simple. Un maestro envia la señal de reloj, y tras cada pulso de reloj envía un bit al esclavo y recibe un bit de éste. Los nombres de las señales son por tanto SCK para el reloj, MOSI para el Maestro Out Esclavo In, y MISO para Maestro In Esclavo Out. Para controlar más de un esclavo es preciso utilizar SS (selección de esclavo).
  • 2C: Es un protocolo síncrono. I2C usa solo 2 cables, uno para el reloj (SCL) y otro para el dato (SDA). Esto significa que el maestro y el esclavo envían datos por el mismo cable, el cuál es controlado por el maestro, que crea la señal de reloj. I2C no utiliza selección de esclavo, sino direccionamiento.

    La imagen hace referencia al montaje realizado para el transmisor de datos del sistema de telemetría del cohete Delta, el cual como se puede observar se basa en el modelo de comunicación de Shannon el cual consiste de una fuente de información, un transmisor, una fuente de ruido en el canal, un receptor y un destino final(este está en la imagen siguiente). En este caso la fuente de información es el sensado, el transmisor el módulo de comunicación que está compuesto de la antena con el circuito impreso, el canal es el aire y las fuentes de ruido que pueden afectar a este son las diferentes tecnologías que trabajan en las bandas de 2.4 y 2.5GHz.

    Como ya se había mencionado anteriormente este montaje se basa en el modelo de comunicación propuesto por Shannon, este montaje hace referencia al receptor el cual en este caso es el segundo módulo de comunicación que está compuesto de la antena con el circuito impreso, y este envía la cadena de datos comunicación serial al destino que es en Matlab para así poder observar estos gráficamente.
